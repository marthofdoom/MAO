# Marth's Alchemy Overhaul (MAO)

A full replacement for Skyrim SE alchemy that removes single-use potions from
loot and vendors and replaces them with permanent, reusable **Flasks** charged
by abstracted **Material Essences** gathered in the field. No ingredient
inventory bloat, no crafting-menu grind: harvest flora and creature drops into
a flat essence currency, unlock effect **Blueprints** by analyzing potions
found in the world, and configure your field kit anywhere.

Built entirely from Linux with no Creation Kit and no xEdit: the plugin is
generated byte-by-byte from Python, Papyrus compiles under Proton wine, and
the native CommonLibSSE-NG layer builds on GitHub Actions — the toolchain
proven by the sibling MRO and MEO projects.

> **Status: design phase.** The spec is written (`Docs/DESIGN.md`); no records
> or code generated yet. See **Roadmap** below and `Docs/INDEX.md`.

## The idea

- **Essences, not ingredients.** Harvesting flora or looting creatures never
  touches your inventory — the native layer intercepts the event and
  increments a three-tier essence currency (Base / Catalyst / Apex) in an
  abstracted Alchemical Pouch.
- **Flasks, not potions.** Your kit holds a small number of permanent Flasks
  (2–6, perk-scaled), each configured with a discovered effect blueprint and
  holding 2–9 charges. Dry flasks refill automatically from your pouch on
  sleep or on a background timer.
- **Blueprints from the world.** Potions stay in enemy drops and boss chests;
  picking one up destroys it and awards a permanent blueprint unlock for that
  effect plus matching essences. Exploration gates capability.
- **Perks buy capacity, not magnitude.** The 13 vanilla/Requiem alchemy perk
  records are overridden in-place to grant flask slots, charge volume, and
  essence efficiency. Standalone MAO never exceeds standard Requiem potion
  caps; over-cap scaling is gated behind MRO integration.
- **Native-first.** Flask state lives in the C++ DLL and serializes to the
  SKSE co-save — no Papyrus save bloat, instant effect application in combat,
  and inventory-count faking so automated potion mods work unpatched.

## Roadmap

- **P0 — the gathering loop (current).** Minimal DLL validating the
  load-bearing assumptions: ingredient acquisition intercepted on every path
  (harvest, loot, container, barter, script), essence counters persisting in
  the SKSE co-save, and the full hook inventory for the rest of the design.
  See `Docs/P0_PLAN.md`.
- **P1 — core.** Flasks (configure / drink / deplete / refill), potion-pickup
  → blueprint interception, the catalog generated from the masters, the
  13-perk override matrix, and an MCM.
- **P2 — coatings & polish.** Weapon coating conversion (Vanguard Coating /
  Corrosive Retention), manual-drink animation handling, field kit UI.
- **P3 — packaging & compat.** FOMOD, the Requiem/LoreRim compatibility
  plugin, and the automated-potion-mod compatibility contract (inventory
  faking, consumption intercept).
- **P4 — MRO integration.** Over-cap magnitude framework gated behind MRO.

## Requirements

SKSE64, SkyUI. Core targets vanilla Skyrim SE + SKSE; a Requiem / LoreRim
compatibility plugin is planned. Developed against the LoreRim toolchain on
Skyrim SE 1.6.
